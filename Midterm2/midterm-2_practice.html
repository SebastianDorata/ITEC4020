Cheat sheet — which one should I use?

Quick decision guide

  • Just need occasional fresh data? → Polling — simplest thing that works.
  • Server → browser live feed, one-way? → SSE — built-in, auto-reconnect.
  • Two-way, low-latency (chat, games)? → WebSockets (or Socket.IO).
  • Fast calls between internal services? → gRPC — contract-first & binary.
  • Full-stack TypeScript, one repo? → tRPC — end-to-end type safety.
  • Async jobs / decouple services? → RabbitMQ — reliable task queue.
  • Huge event streams, replayable? → Kafka — distributed event log.

Networking basics

localhost, loopback & ports

  • localhost refers to the computer you are currently using — it always points back to your own machine.
  • The loopback address is 127.0.0.1, the IP address a machine uses to refer to itself.
  • A port number identifies a specific process or service on a device — think of it as a "door" into the computer (e.g. port 3000 for your Express app).
  • IP addresses uniquely identify devices on a network, letting client and server locate and communicate with each other.
  • Binding a server to 127.0.0.1 / localhost means only that machine can connect. Binding to 0.0.0.0 accepts connections from any network interface, which is what lets external machines reach it.
